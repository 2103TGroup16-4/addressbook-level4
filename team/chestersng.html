PROJECT: CLInic

Overview

CLInic is a desktop application made for clinics. The CLI in CLInic stands for Command Line Interface, which is the main method to use this application. CLInic also has a Graphical User Interface (GUI) to display information in a structured and user-friendly manner. This application uses Java as the programming language and uses JavaFX, HTML and CSS for the GUI.

My task in CLInic is to implement a Patient Queue Management System, as well as to integrate the changes of my team to ensure CLInic works after every enhancement or change.

The project scope for CLInic is to morph the given addressbook-level4 into a clinic management application that allows users to manage patient information and medicine information and additionally manage the clinic queue.

The team consists of four year-2 and one year-3 student in School of Computing, NUS.

Queue Management

Listing all Patients in Patient Queue : queue

Lists all patients in queue waiting to see the doctor.
Alias: q
Format: queue

Registering Patient to Queue : register

Registers a new patient that comes to the clinic.
Alias: reg
Format: register <INDEX> or register <IC Number> [coming soon]

Example:

  • register 3
    Register the patient into the 3rd index.

Removing a patient from queue : remove

Removes someone from the queue if he/she leaves before being served.
Alias: rem
Format: remove <INDEX> or remove <IC Number> [coming soon]

Examples:

  • remove 5
    Remove the 5th patient in the queue.

  • remove S9382839X
    Remove patient with IC Number S9382839X from the queue.

Inserting a patient to queue : insert

Inserts a patient at specified position in the queue.
Alias: ins
Format: insert <INDEX> <POSITION> or insert <IC Number> <POSITION> [coming soon]

Examples:

  • insert 4 5
    Insert the 4th person of the address book into the 5th index of the queue.

  • insert S9123456A 5
    Insert the patient with that IC number to the 5th index of the queue.

Serving the first patient in the queue : serve

Serves the patient first in queue.
Alias: ser
Format: serve

Finish serving the Current Patient : finish

Transfers the current patient to the Served Patient Queue.
Format: finish

Patient Queue Management System

The Patient Queue Management System consists of three main classes: PatientQueue, CurrentPatient and ServedPatientQueue. A patient that registers to see the doctor is put into the PatientQueue. When it is the patient’s turn to see the doctor, he will become the CurrentPatient. After he is done with seeing the doctor, he will be put into the ServedPatientQueue to await payment and document processing.

Given below is an example usage scenario and how the Patient Queue Management System is used in each step.

Step 1. Three patients visits the clinic, the user executes the register command for each patient.

Figure 1. Registering 3 patients

Step 2. It is a patient’s turn for consultation. The user executes the serve command to serve the first patient in the queue. The patient is now the CurrentPatient.

Figure 2. Serve patient

Step 3. The user executes various commands to add information to the CurrentPatient. These information will be used for document processing later on.

Step 4. The CurrentPatient is done with the consultation. The user executes the finish command to transfer this patient to the ServedPatientQueue. Afterwards, the user executes serve to get the next patient.

Figure 3. Finish serving patient

Step 5. The user executes various commands to generate necessary documents for the served patient.

Step 6. The user finally executes payment command to generate the receipt and ends the patient’s visit to the clinic, removing him from the ServedPatientQueue.
